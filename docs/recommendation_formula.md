# Алгоритм расчета рекомендаций по трекам и карточкам результата

## 1. Базовый скоринг теста
В тесте 6 направлений:
1. Технологии и инженерия
2. Исследование и анализ
3. Творчество и медиа
4. Коммуникации и влияние
5. Помощь людям и обучение
6. Практика и организация

Источники баллов:
- шкальные вопросы: 1–5 баллов по направлению
- forced choice: 2 балла выбранному направлению
- сценарные вопросы: 3 балла выбранному направлению

Итог:
- считаем суммарный score по каждому из 6 направлений
- сортируем направления по убыванию
- получаем top-3 направления

## 2. Нормализация в проценты
`percent = round(score / max_score_for_direction * 100)`

Где `max_score_for_direction` зависит от количества вопросов по направлению.

## 3. Логика выбора рекомендованного трека
Трек — интерпретационный слой поверх результатов.

### 3.1 Основное правило
- Технологии и инженерия → Академическое образование
- Исследование и анализ → Академическое образование
- Творчество и медиа → Общее образование
- Коммуникации и влияние → Общее образование
- Помощь людям и обучение → Общее образование
- Практика и организация → Профессиональное образование

### 3.2 Альтернативный трек
- Технологии и инженерия → Профессиональное образование
- Исследование и анализ → Общее образование
- Творчество и медиа → Профессиональное образование
- Коммуникации и влияние → Академическое образование
- Помощь людям и обучение → Академическое образование
- Практика и организация → Академическое образование

## 4. Логика выбора текста карточки
После определения 1-го места:
1. берем шаблон карточки соответствующего направления
2. подставляем:
   - название направления
   - percent выраженности
   - рекомендованный трек
   - «Почему этот трек может подойти»
   - «Подходящие направления»
   - альтернативный трек
   - примеры учреждений

## 5. Правила для top-2 и top-3
Практический вариант:
- 1 место — полная карточка
- 2 и 3 места — сокращенные карточки

## 6. Псевдокод
```text
scores = calculate_scores(answers)
ranking = sort_desc(scores)
primary = ranking[0]
secondary = ranking[1]
tertiary = ranking[2]

card = card_template[primary.direction]
percent = normalize(primary.score)

result = {
  primary_direction: primary.direction,
  primary_percent: percent,
  recommended_track: track_map[primary.direction],
  alternative_track: alt_track_map[primary.direction],
  card_text: card
}
```

## 7. Ограничения
Модель подходит для прототипа и пилота:
- прозрачна и легко объясняется,
- но тест измеряет склонности, а не маршрут напрямую.

Итоговые формулировки должны быть мягкими: «может подойти», «что еще можно рассмотреть».
